# Decision Log v0.7

## 2026-05-06 — v0.7 통합 결정

기준 패키지는 v0.7.0 Python 계산 인프라를 사용한다. reference generation의 24절기 1950~2030 reference, 데이터 생성 방법론, 외부 기준 케이스 자정경계 검증 케이스를 통합한다.

## 확정 정책

1. 스킬은 계산 전용이다. 사주 해석은 본체 에이전트가 담당한다.
2. 양력 생년월일만 지원한다.
3. 음력/윤달은 v1.x backlog로 둔다.
4. 국내 출생만 전제한다.
5. 출생지는 사용자에게 묻지 않는다.
6. 내부 기본 출생지는 서울특별시다.
7. 대한민국 법정시/서머타임을 자동 보정한다.
8. 서울 경도 보정 -32분을 적용한다.
9. 진태양시/시태양시/균시차는 적용하지 않는다.
10. 연주는 입춘 절입시각 기준이다.
11. 월주는 12개 월 경계 절기 절입시각 기준이다.
12. 일주는 입력 양력 일자의 일진을 그대로 사용한다.
13. 子시는 23:00~00:59지만, 子시여도 다음날 일주로 넘기지 않는다.
14. 시주 천간은 입력 양력일의 일간 기준이다.
15. 24절기 라벨과 황경이 충돌하면 황경을 우선 신뢰한다.

## reference generation v1.0에서 통합한 것

- `references/jeolgi/{YYYY}.json` 1950~2030 전체 구조.
- KASI 2000~2027 + skyfield/DE421 1950~1999, 2028~2030 방법론.
- skyfield `ecliptic_latlon('date')` 필수 주의사항.
- KASI 원본 정정 이력 4건.
- 2015 하지 review_note.
- 외부 기준 케이스 자정경계 검증 케이스 2건.
- tests 구조와 expected output 방향.

## v0.7 통합 중 추가 발견

reference generation v1.0 reference의 `2000.json`에 24개가 아니라 23개만 있고, `우수`가 빠져 있으며 `2000-02-19 17:33` 항목이 `입춘/315°`로 들어가 있었다. 이 시각은 우수에 해당하므로 다음처럼 보정했다.

- true 입춘: `2000-02-04 21:40`, `sunLongitude=315`, source `skyfield_de421`.
- 기존 `2000-02-19 17:33`: `우수`, `sunLongitude=330`으로 재분류.

이 보정 후 1950~2030 전체 reference와 runtime cache가 81년 × 24개 = 1,944 entries로 검증된다.
