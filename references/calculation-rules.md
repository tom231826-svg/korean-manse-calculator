# 계산 규칙 v0.7

## 1. 범위와 입력

- 지원 범위: 1950-01-01 ~ 2030-12-31.
- 입력 달력: 양력만 지원.
- 출생지: 사용자에게 받지 않고 서울특별시를 기본값으로 둔다.
- 해외 출생, 음력/윤달, 진태양시/균시차는 v0.7 미지원.

## 2. 시간 보정

사용자 입력 시각은 당시 대한민국 법정 시계 시간으로 간주한다.

```text
입력 법정시각
→ zoneinfo:Asia/Seoul로 과거 표준시/서머타임을 fixed KST(+09:00)로 환산
→ 서울 경도 보정 -32분 적용
→ 년주/월주/시주 판정 기준시각
```

서울 -32분은 경도 보정 성격이다. 균시차는 적용하지 않는다.

## 3. 일주 정책: 입력 양력일 고정

v0.7의 일주 기준은 다음과 같다.

```text
일주 = 사용자가 입력한 양력 일자의 일진
```

보정시각이 전날/다음날로 넘어가도 일주 날짜를 바꾸지 않는다. 子시여도 다음날 일주로 넘기지 않는다.

외부 기준 케이스 검증 케이스:

| 입력 | 보정시각 | 결과 |
|---|---:|---|
| 1990-01-01 23:30 | 1990-01-01 22:58 | 일주 병인 |
| 1990-01-02 00:30 | 1990-01-01 23:58 | 일주 정묘 |

## 4. 연주

입춘 절입시각 기준.

```text
보정시각 < 해당년도 입춘 → 전년도 세차
보정시각 >= 해당년도 입춘 → 해당년도 세차
```

1984년을 甲子년으로 두고 60갑자 순환을 적용한다.

## 5. 월주

월주는 음력월이 아니라 24절기 중 12개 월 경계 절기 기준이다.

| 월지 | 시작 절기 | 황경 |

| 卯 | 경칩 | 345 |

| 巳 | 입하 | 45 |
| 午 | 망종 | 75 |
| 未 | 소서 | 105 |
| 申 | 입추 | 135 |

## 6. 시주

- 시지 기준은 보정시각.
- 子시 = 23:00~00:59.
- 시주 천간은 입력 양력일의 일간 기준으로 오자둔 표를 적용한다.
- 시간 미상은 시주를 `null`로 둔다. 00:00으로 가정하지 않는다.

## 7. 황경 Fallback 규칙

절기 entry의 `name`과 `sunLongitude`가 모순되면 `sunLongitude`를 우선 신뢰한다. 라벨 오류는 드물지만 2007년 대서/대설 사례처럼 실제 월주 오류로 이어질 수 있으므로 방어 규칙으로 고정한다.

| 황경 | 절기 | 황경 | 절기 | 황경 | 절기 |
|---:|---|---:|---|---:|---|
| 0 | 춘분 | 120 | 대서 | 240 | 소설 |
| 15 | 청명 | 135 | 입추 | 255 | 대설 |
| 30 | 곡우 | 150 | 처서 | 270 | 동지 |
| 45 | 입하 | 165 | 백로 | 285 | 소한 |
| 60 | 소만 | 180 | 추분 | 300 | 대한 |
| 75 | 망종 | 195 | 한로 | 315 | 입춘 |
| 90 | 하지 | 210 | 상강 | 330 | 우수 |
| 105 | 소서 | 225 | 입동 | 345 | 경칩 |

## 8. 24절기 데이터 방법론

- KASI 직접 구간: 2000~2027.
- skyfield + NASA/JPL DE421 보완 구간: 1950~1999, 2028~2030.
- skyfield 재현 시 `ecliptic_latlon('date')`를 반드시 사용한다. J2000 기본 frame 사용 금지.
- reference generation 방법론 기준 KASI ground truth 615개와 cross-validation 99.5% (612개) 1분 이내 일치.

## 9. 데이터 정정 이력

- 2007-12-07: 대서 라벨 오류 → 대설 정정 (`sunLongitude=255`).
- 2019-01-20: `17:60` → `18:00`.
- 2011-01-21 대한: 1일 typo 의심 → skyfield `2011-01-20 19:18`.
- 2011-11-08 입동: 6시간 차이 → skyfield `2011-11-08 03:34`.
- 2015 하지: KASI와 skyfield 20분 차이, KASI 채택 및 `review_note`.
- 2000 입춘/우수: v0.7 통합 검증 중 우수 누락/입춘 날짜 오염 발견, reference와 runtime cache 보정.
